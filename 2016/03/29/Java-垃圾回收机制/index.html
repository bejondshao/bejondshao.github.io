Java Garbage Collection(GC)
GCе°ұжҳҜз§»йҷӨжңӘиў«еј•з”Ёзҡ„еҜ№иұЎ(еһғеңҫеҜ№иұЎ), жё…зҗҶеҮәеҶ…еӯҳжқҘеҲҶй…Қе…¶д»–ж–°еҜ№иұЎ. Javaеһғеңҫеӣһж”¶жҳҜиҮӘеҠЁзҡ„, иҝҷеҹәжң¬дёҚйңҖиҰҒзЁӢеәҸе‘ҳз®ЎзҗҶ, еҢәеҲ«дәҺC C++зӯүиҜӯиЁҖ, йңҖиҰҒжүӢеҠЁеҲҶй…ҚеҶ…еӯҳ, жүӢеҠЁйҮҠж”ҫ. жүҖд»ҘиҝҷдҪҝеҫ—JavaзЁӢеәҸе‘ҳеңЁеҶ…еӯҳдҪҝз”Ёж—¶жҳҫеҫ—иӮҶж— еҝҢжғ®, еҜ№иұЎеӣһж”¶д№ҹж— жі•жҸ’жүӢ, жүҖд»ҘJavaиҮӘеҠЁеӣһж”¶еһғеңҫжңәеҲ¶д№ҹжҲҗдёәJavaйЎ№зӣ®зҡ„дёҖеӨ§иҜҹз—…. йЎ№зӣ®еҲ°дёҖе®ҡи§„жЁЎеҠҝеҝ…дјҡжңүжҖ§иғҪй—®йўҳ. еӣ жӯӨеҸ‘зҺ°жҖ§иғҪй—®йўҳ, и§ЈеҶіжҖ§иғҪй—®йўҳжҳҜдёҖйЎ№жңүз”Ёзҡ„жҠҖиғҪ.

GCеңЁеҗҺеҸ°иҝҗиЎҢ, жҹҘжүҫжүҖжңүзҡ„еҜ№иұЎ, жүҫеҲ°еһғеңҫеҜ№иұЎ, з„¶еҗҺеҲ йҷӨе®ғд»¬. GCдёҖиҲ¬еҢ…жӢ¬иҝҷдёүжӯҘ:

  1. ж Үи®°: е°ҶеһғеңҫеҜ№иұЎж Үи®°жҲҗеҸҜиў«еӣһж”¶зҠ¶жҖҒ.
  2. еҲ йҷӨ: е°ұжҳҜеҲ йҷӨе•Ұ.
  3. еҺӢзј©: еңЁеҲ йҷӨеһғеңҫеҜ№иұЎеҗҺ, дёәдәҶжӣҙеҘҪзҡ„жҖ§иғҪ, дјҡе°Ҷеӯҳжҙ»зҡ„еҜ№иұЎйҮҚж–°ж•ҙзҗҶеҲ°дёҖиө·, иҝҷж ·дјҡз©әеҮәжҲҗеқ—зҡ„еҶ…еӯҳз©әй—ҙ.

дёҠиҝ°GCиҝҮзЁӢжңүй—®йўҳ, жңүд»Җд№Ҳй—®йўҳ?

  1. еҫҲеӨҡеҲҡеҲӣе»әзҡ„еҜ№иұЎе°ұжҳҜжңӘиў«дҪҝз”Ёзҡ„.
  2. еңЁеҫҲеӨҡGCе‘ЁжңҹйҮҢзҡ„еҜ№иұЎеҫҲеҸҜиғҪиў«жңӘжқҘе…¶д»–GCе‘ЁжңҹдҪҝз”Ё. е°ұжҳҜиҜҙеҫҲеёёз”Ёзҡ„еҜ№иұЎеҸҜиғҪеңЁжңӘжқҘиҝҳдјҡиў«дҪҝз”Ё, еҰӮжһңеҲ йҷӨдәҶеІӮдёҚеҸҜжғң.

жңүд»Җд№ҲеҘҪж–№жі•? Javaеӣһж”¶жңәеҲ¶еҲ©з”ЁYong Genзҡ„Minor GCе’ҢOld Genзҡ„Major GCжқҘеӨ„зҗҶиҝҷдёӨдёӘй—®йўҳ. Minor GCеҸӘдјҡеңЁEdenж»ЎдәҶзҡ„ж—¶еҖҷжү§иЎҢ, жүҖд»ҘеҸҜд»ҘйҒҝе…Қй—®йўҳ1. Major GCдјҡеңЁOld Genж»ЎдәҶжү§иЎҢ, жүҖд»ҘдјҡйҒҝе…Қй—®йўҳ2.

Java GC з§Қзұ»
GCдёҖе…ұжңүдә”з§Қ, жҲ‘д»¬еҸҜд»ҘйҖҡиҝҮй…ҚзҪ®жқҘдҪҝз”ЁдёҚеҗҢзҡ„жңәеҲ¶

  1. Serial GC(-XX:+UseSerialGC): дёІиЎҢGCдҪҝз”Ёж Үи®°-еҲ йҷӨ-еҺӢзј©зҡ„жЁЎејҸжқҘиҝӣиЎҢMinor GCе’ҢMajor GC. дёІиЎҢGCдёҖиҲ¬йҖӮз”ЁдәҺе®ўжҲ·з«Ҝ, жҜ”еҰӮдёӘдәәз”өи„‘, йқһйӣҶзҫӨзҡ„жңҚеҠЎеҷЁ. йҖӮз”ЁдәҺCPUдёҚеӨӘејәжӮҚзҡ„ж—¶еҖҷ.
  2. Parallel GC(-XX:+UseParallelGC): е№¶иЎҢGCзӣёеҜ№дәҺдёІиЎҢGC, жҳҜз”ЁNдёӘзәҝзЁӢжү§иЎҢMinor GC, NдёҖиҲ¬е’ҢCPUж ёеҝғж•°зӣёеҗҢ, NеҸҜд»ҘйҖҡиҝҮ-XX:ParallelGCThreads=nжқҘжҺ§еҲ¶. Nе№¶дёҚжҳҜи¶ҠеӨ§и¶ҠеҘҪ, дёҚиҰҒи¶…иҝҮCPUж ёеҝғж•°.
  3. Parallel Old GC (-XX:+UseParallelOldGC): иҝҷз§Қж–№ејҸд№ҹжҳҜе№¶иЎҢGC, еҢәеҲ«еүҚиҖ…жҳҜ, иҝҷз§Қж–№ејҸеңЁYoung Genе’ҢOld GenйғҪдјҡдҪҝз”Ёе№¶иЎҢжңәеҲ¶.
  4. Concurrent Mark Sweep (CMS) Collector (-XX:+UseConcMarkSweepGC): CMSеҸ«ж—¶е®һж Үи®°жё…зҗҶжңәеҲ¶(жҲ‘иө·зҡ„, й«ҳеӨ§дёҠеҗ§, е“Ҳе“Ҳе“Ҳ). е°ұжҳҜеңЁOld GenйҮҢ, ж Үи®°дәҶеһғеңҫеҜ№иұЎ, дёҚзӯүOld Genж»Ў, е°ұејҖе§Ӣеӣһж”¶. иҝҷжңүдёҖдёӘеҘҪеӨ„жҳҜжҜҸж¬Ўеӣһж”¶зҡ„еҜ№иұЎеҫҲе°‘, stop-the-worldзҡ„ж—¶й—ҙеҫҲзҹӯ, дҪҺе»¶иҝҹ. иҝҷеңЁе“Қеә”йҖҹеәҰиҰҒжұӮй«ҳзҡ„еә”з”ЁдёҠдҪҝз”Ё, жҜ”еҰӮиҙӯзү©зҪ‘з«ҷ, еҰӮжһңзҪ‘з«ҷжҡӮеҒңж—¶й—ҙд№…дәҶ, е®ўжҲ·еҸҜиғҪе°ұе…ізҪ‘йЎөдёҚд№°дәҶ, еҺ»еҲ«е®¶еҺ»д№°дәҶ, иҝҷе“ӘиЎҢ. дҪҶжҳҜ, иҝҷз§Қж–№ејҸ, жғіжғіе°ұжҳҺзҷҪ, иӮҜе®ҡйңҖиҰҒејәжӮҚзҡ„зЎ¬д»¶ж”ҜжҢҒ, йў‘з№Ғзҡ„GCйғҪдјҡжҳҜеҫҲеӨ§зҡ„ејҖй”Җ. CMSеңЁYoung Genзҡ„ж–№ејҸе’ҢParallel GCдёҖж ·, жҲ‘д»¬еҸҜд»Ҙз”Ё-XX:ParallelCMSThreads=nжқҘйҷҗеҲ¶еӣһж”¶иҝӣзЁӢж•°.
  5. G1 Garbage Collector (-XX:+UseG1GC): еһғеңҫз¬¬дёҖ, д№ҹеҸ«G1, иҝҷз§ҚжЁЎејҸеңЁjava 7д»ҘдёҠзҡ„зүҲжң¬еҸҜз”Ё, д»Һй•ҝиҝңжқҘзңӢ, е®ғиҰҒжӣҝд»ЈCMSеӣһж”¶жЁЎејҸ. G1еӣһж”¶д№ҹжҳҜе№¶иЎҢзҡ„, ж—¶е®һзҡ„, е№¶дё”жҳҜеўһйҮҸеҺӢзј©зҡ„.
    иҝҷз§Қж–№ејҸдёҚиҖғиҷ‘Young GenиҝҳжҳҜOld Gen, е®ғе°ҶHeapеҲҶжҲҗеҮ зӯүд»ҪеҢәеҹҹ. еҪ“JVMи°ғз”ЁGCж—¶, е®ғе…Ҳжё…зҗҶеӯҳжҙ»еҜ№иұЎе°‘зҡ„еҢәеҹҹ, д№ҹе°ұжҳҜе“ӘйҮҢеһғеңҫеӨҡжү«е“ӘйҮҢ. жӣҙеӨҡдҝЎжҒҜGarbage-First Collector Oracle Documentation.
